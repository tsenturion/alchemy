# Alchemy

Alchemy - статический веб-справочник по алхимии Skyrim на русском языке. Он показывает ингредиенты, их эффекты и помогает подбирать совместимые компоненты для зелий или ядов.

## Возможности

- поиск ингредиентов по названию;
- фильтр по эффекту через меню "Выбрать Эффект" или клик по эффекту в таблице;
- выбор до 3 ингредиентов;
- подсветка положительных, отрицательных и смешанных эффектов;
- таблица возможных сочетаний для выбранных ингредиентов;
- подключение и отключение наборов данных из DLC, Creation Club, CACO и других дополнений.

## Запуск локально

Проект не требует сборки и пакетного менеджера. Нужен только локальный HTTP-сервер, потому что браузер загружает JSON-файлы через `fetch`.

В Windows из корня репозитория:

```powershell
py -m http.server 8000
```

Если команда `py` недоступна:

```powershell
python -m http.server 8000
```

После запуска открой:

```text
http://localhost:8000/
```

Прямое открытие `index.html` из файловой системы может не загрузить данные из-за ограничений браузера.

## Структура проекта

```text
.
|-- index.html          # Разметка страницы
|-- styles.css          # Стили интерфейса
|-- src/
|   `-- app.js          # Логика загрузки данных, фильтров и подбора сочетаний
`-- data/
    |-- data.json       # Основной набор ингредиентов
    |-- effects.json    # Справочный список эффектов
    |-- positive_negative_data.json
    |-- DLC/
    |-- Creation Club/
    |-- CACO/
    `-- Skyrim Extended Cut - Saints and Seducers/
```

## Формат данных

Основные файлы ингредиентов называются `data.json` и содержат массив объектов:

```json
[
  {
    "name": "Алый корень Нирна",
    "effects": [
      "Невидимость",
      "Сопротивление магии",
      "Урон запасу сил",
      "Урон здоровью"
    ],
    "effect_state": 0
  }
]
```

Интерфейс использует поля `name` и `effects`. Поле `effect_state` может присутствовать в данных, но логика страницы на него не опирается.

Классификация эффектов хранится в `positive_negative_data.json`:

```json
{
  "positive_effects": ["Восстановление здоровья"],
  "negative_effects": ["Урон здоровью"]
}
```

Файлы `positive_negative_data.json` внутри дополнений необязательны. Если они есть, приложение учитывает их при объединении данных.

## Добавление нового набора данных

1. Создай папку внутри `data/` и положи в нее `data.json` с тем же форматом, что у основного набора.
2. Если набор добавляет новые эффекты, добавь рядом `positive_negative_data.json`.
3. Если автоматическое обнаружение папки недоступно в окружении запуска, добавь путь к `data.json` в `FALLBACK_ADDON_DATA_PATHS` в `src/app.js`.
4. Если нужно красивое название в меню "Дополнения", добавь соответствие в `ADDON_DISPLAY_NAMES` в `src/app.js`.

## Публикация

Проект можно публиковать как обычный статический сайт, например через GitHub Pages из корня репозитория. При запуске на GitHub Pages приложение пытается автоматически найти вложенные `data.json` в папке `data/` через GitHub API, а при неудаче использует список запасных путей из `src/app.js`.
